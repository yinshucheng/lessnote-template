# Evening - 每日工作总结

当用户调用 /evening 时，执行以下流程：

## 1. 自动信息加载

- 调用 `flow_generate_daily_summary` 获取今日番茄和完成情况
- 读取当前周文档，找到今日的时间线记录
- 读取本周焦点

## 2. 更新周文档时间线

在今日时间线下补充实际完成情况：

```markdown
### MM-DD（周X）
- 计划：xxx / xxx / xxx
- 完成：xxx ✅ / xxx ❌（原因）
- 番茄：X个（Y分钟）
- （如有洞察/问题，记录在这里）
```

## 3. 本周焦点进度检查

对比本周焦点和这几天的实际投入：

- 焦点1：本周投入了几个番茄？有什么进展？
- 焦点2：同上
- ...

如果某个焦点连续2天无投入，直接问："xxx 已经2天没动了，是遇到了阻碍还是优先级变了？"

## 4. 工作节奏与健康提醒

### 高强度工作后提醒休息
- 今日番茄 >= 6个："今天产出很高！但也要充电"
- 连续3天番茄 >= 4个："连续高强度工作3天了，安排一些让自己开心的事"

### 娱乐缺失提醒
- 本周无娱乐/运动记录：
  - 周三前：轻提醒
  - 周四后：强提醒

### 提醒原则
- 娱乐不是浪费时间，是让高效工作可持续的前提

## 5. 简短回顾（对话中进行）

问用户：
- 今天最有价值的一件事是什么？
- 有没有什么需要记到项目文档里的决策/发现？

## 6. 明日预告

- 提醒明日有DDL的灰犀牛事项
- 特殊日期提醒（预约的会议、截止日期等）

## 6.5 文档健康检查（月底触发）

**每月最后一个工作日**的 /evening 额外执行：

| 频率 | 检查项 |
|------|--------|
| **月底** | 年度目标文档 — 进度是否需要刷新 |
| **月底** | 项目全景文档 — 有无新项目需加入、有无项目该归档 |
| **月底** | 停滞项目清理 — 超过4周无进展的项目 → 归档或标注"暂停" |

## 7. 作息提醒

- 21:00后：提醒回家
- 22:00后：禁止新番茄
- 主动问："今晚计划几点睡？"

## 8. 同步到团队日报（如已配置 iie-worklog）

询问用户是否需要同步到团队日报。如果确认：

1. 从今日周文档时间线中提取**工作相关**内容
2. **排除**：个人事务、健康、理财、感情等非工作内容
3. 生成团队日报格式：
   - 今日工作（已完成 / 进行中）
   - 关键技术决策或发现
   - 遇到的问题 / 需要协助的事项
   - 推荐的工具/文章/想法（如有）
   - 明日计划
4. 写入团队日报仓库对应日期文件
5. git add + commit（在个人分支）
6. 询问用户是否 rebase + push 到共享分支

**注意**：
- 日志内容面向团队，语言要专业简洁
- 如果用户说"不需要"，直接跳过

## 强制规则

- ❌ 22:00后禁止启动新番茄
- ✅ 主动检查本周焦点投入
- ✅ 提醒工作节奏和娱乐安排
- ✅ 提醒灰犀牛
- ✅ 提醒作息
- ✅ 询问是否同步团队日报

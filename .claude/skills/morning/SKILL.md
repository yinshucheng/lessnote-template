# Morning - 每日工作启动

当用户调用 /morning 时，执行以下流程：

## 1. 自动信息加载（AI执行，用户无需操作）

- 读取当前周文档：`weekly/` 目录下本周文件
- 读取灰犀牛事项：`backlog/灰犀牛事项.md`（筛选7天内DDL）
- 读取超期任务：调用 `flow_get_overdue_tasks`（如已配置 Vibeflow）
- 读取任务管理系统今日任务（如已配置）
- 调用 `flow_generate_daily_summary` 获取昨日番茄数据（如已配置）
- **读取团队日报动态**（如已配置 iie-worklog）：扫描团队成员昨日日报，提取与自己工作相关的信息（如依赖项进展、可复用的成果、需要对齐的事项）

## 2. 检查本周焦点

读取周文档顶部的「本周焦点」，检查：

- **投入不足预警**：检查本周焦点中是否有连续2天无进展的事项，主动提醒
- **产出检查**：不问"你在推进吗"，问"这件事本周的交付物是什么，现在完成了多少"
- **公开输出检查**：如果目标中包含公开输出（博客/分享/文章），提醒检查进展
- **注意**：不限制并行数量，多线并行是优势。真正要挑战的是：刷手机、想象困难不动手、思虑过度

## 3. 团队动态速览（如已配置 iie-worklog）

快速扫描团队日报仓库中其他成员的最新日报，提取：

- **与我相关的进展**：别人提到的、和我工作有交集的内容
- **可复用的成果**：别人发现的工具/方案/文档，对我可能有用
- **需要对齐的事项**：有依赖关系的工作，需要今天沟通确认的

格式：
```
团队动态（与你相关）：
- [同事A] 完成了 XXX，和你的 YYY 有关联
- [同事B] 提到遇到 ZZZ 问题，你之前做过类似的
```

如果没有与自己直接相关的内容，跳过这一步，不要硬凑。

## 4. 工作节奏与健康提醒

基于历史数据判断工作节奏：

- **连续高强度**：如果昨日番茄 >= 6个，或连续3天 >= 4个，提醒注意休息
- **娱乐缺失**：如果本周时间线中没有任何娱乐/运动记录，且已过周三，提醒充电
- **晚睡预警**：如果用户提到昨晚睡得晚，提醒注意但不批判

## 5. 口头简报（不写入文档）

直接在对话中告诉用户：

```
本周焦点进度：
1. xxx — 进展/待做
2. xxx — 进展/待做
...

灰犀牛（7天内）：
- xxx - DDL xx-xx

超期任务：
- xxx（超期X天）

团队动态（与你相关）：
- [同事X] ...

工作节奏：昨日X个番茄，本周累计Y个
```

## 6. 确认今日计划

- 询问用户今天打算做什么（结合用户 /morning 时传入的参数）
- 检查每个任务是否有时间盒
- 如果计划和本周焦点不一致，给出提醒但不强制
- 确认后，询问是否启动第一个番茄

## 7. 写入周文档 + 团队日报（可选）

**写入周文档**：在今日时间线区域追加计划。

```markdown
### MM-DD（周X）
- 计划：xxx / xxx / xxx
```

**写入团队日报（如已配置 iie-worklog）**：
询问用户是否将今日计划同步到团队日报。如果确认：
- 从今日计划中提取**工作相关**内容（排除个人事务）
- 写入团队日报的"今日计划"部分
- git add + commit（在个人分支上）

## 7.5 周度文档健康检查（周一或周末触发）

**每周一或周末**的 /morning 额外执行（每周只执行一次）：

| 检查项 | 做什么 |
|--------|--------|
| `projects/README.md`（项目全景） | 有无新项目需加入、有无项目该归档 |
| 活跃项目文档 | 状态是否需刷新 |
| `backlog/想法收集.md`（如存在） | 超过4周未决策的想法 → 提醒处理 |
| 停滞项目 | 超过4周无进展的活跃项目 → 建议归档或标注"暂停" |

## 强制规则

- ❌ 不允许没有时间盒的任务
- ✅ 主动提醒灰犀牛事项
- ✅ 本周焦点投入不足时主动提醒
- ✅ 今日计划与本周焦点冲突时主动提醒
- ✅ 基于历史数据提醒工作节奏和娱乐安排
- ✅ 周一执行周度文档健康检查

## 关于周文档

如果是本周第一次 /morning（周一），需要创建新的周文档，格式见下方模板。

### 新周文档模板

```markdown
# 2026年第N周工作记录

**日期：** MM-DD 至 MM-DD

## 本周焦点

> 每件事必须有本周的具体产出。不是"推进一下"，是"这周要交付什么"。

| # | 事项 | 本周交付物 | 关联 |
|---|------|-----------|------|
| 1 | xxx | 具体产出物 | 项目/目标 |

---

## 时间线

### MM-DD（周X）
- 做了什么
- （遇到问题时展开记录）
```

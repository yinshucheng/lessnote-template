# Sync - 工作日中间同步

当用户调用 /sync 时，执行以下流程：

## 1. 信息加载（AI 自动执行）

- 读取 Vibeflow 今日任务和番茄记录
- 读取今日启动清单（周文档）
- 检查当前时间，计算剩余工作
生成当前进展摘要：

```
**时间**：当前 HH:mm，时间

## 2. 进展汇总
今日已工作 X 小时，剩余 Y 小时

**今日 TOP 3 进展：**
1. [任务1] 2. [任务2] - ...
3. [任务3] - ...

**番茄统计：**
- 已完成：X 个
- 专注时长：Y 分钟
```

## 3. 评估与调整

询问用户：
-- ✅完成 / 🟡进行中 / ❌未开始
 今天的 TOP 3 还合理吗？需要调整吗？
- 有新的紧急事项吗？
- 有什么卡点需要处理？

根据剩余时间，建议调整：
- 如果时间不够，建议砍掉或推迟某个任务
- 如果有新紧急事项，建议替换优先级最低的任务
- 提醒：每日最多 3 个任务

## 4. 记录到文档

更新周文档中当天的记录：
- 添加/更新完成情况
- 记录任务调整（如有）
- 记录重要进展或问题

## 5. 后续计划

确认下一个番茄要做什么：
- 从 TOP 3 中选择下一个任务
- 设定时间盒
- 启动番茄（如果用户需要）

## 使用场景

- 午饭后回来，重新 sync 状态
- 完成一个大任务后，重新评估后续
- 被打断后，恢复工作节奏
- 感觉有点迷失方向时

## 与其他 Skills 的关系

| Skill | 时机 | 侧重点 |
|-------|------|--------|
| `/morning` | 开始工作 | 设定今日方向 |
| `/sync` | 工作中间 | 评估调整，保持方向 |
| `/evening` | 结束工作 | 总结反思，规划明日 |
| `/checkpoint` | 番茄结束 | 记录单任务进展 |

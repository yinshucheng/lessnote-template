# Focus - 工作进行中监控

当用户调用 /focus 或开始一个任务时，执行以下流程：

## 0. 番茄状态检查（首先执行）

**第一步：检查是否已有番茄在进行中**

调用任务管理 MCP（如 Vibeflow）检查当前番茄状态：

1. **如果已有番茄在进行中**：
   - 显示当前番茄信息（任务名、已用时间）
   - 询问用户是否继续当前任务，还是切换任务

2. **如果没有番茄在进行中**：
   - 根据用户传入的任务描述（argument），搜索匹配的任务
   - 确认后启动番茄
   - 如果没有找到匹配任务：创建新任务，然后启动番茄

---

## 1. 任务启动前强制结构化

在任务开始前，强制用户回答5个问题（5分钟内完成，不要变成拖延）：

```
1. 我要解决什么问题？（1句话）
2. 我目前的思路是什么？（最多3点）
3. 我需要谁的什么输入？（具体人+具体问题）
4. 完成的标准是什么？（"够用"的定义）
5. 第一个可验证的里程碑是什么？（15分钟内能得到反馈的最小动作）
```

**特别关注第3点** — 强制识别沟通需求
**特别关注第5点** — 强制建立反馈闭环（见下方原则）

## 2. 时间盒设定

- 询问："这个任务你计划花多少个番茄？"
- 如果超过3个番茄，要求拆分成子任务
- 每个子任务独立设定时间盒

## 3. 执行中：反馈闭环加速

### 核心原则：像 Agent 一样工作

人的工作方式容易陷入长链条纯思考（Chain of Thought），不断推导却不验证。
更高效的方式是像 ReAct Agent 一样：**思考 → 行动 → 观察 → 调整**，每一步都从环境获取真实反馈。

```
低效模式（纯思考链）：
  想 → 想 → 想 → 想 → 做 → 发现方向错了

高效模式（ReAct 反馈闭环）：
  想 → 做一小步 → 看结果 → 调整方向 → 做下一步 → 看结果 → ...
```

### 反馈闭环检查点

**每 15 分钟或每个小步骤完成后，问：**

1. **我刚才的假设被验证了吗？** — 是 → 继续；否 → 调整方向
2. **我现在掌握了什么新信息？** — 行动产生的信息比思考产生的更有价值
3. **下一个最小可验证步骤是什么？** — 不要规划太远，只看下一步

### 识别"纯思考陷阱"的信号

| 信号 | AI 应提醒 |
|------|----------|
| 用户连续思考超过 15 分钟没有任何产出 | "你现在能做一个最小实验来验证你的想法吗？" |
| 用户在文档/方案上反复修改措辞 | "这个方案够不够用？能不能先让别人看一眼给反馈？" |
| 用户说"我再想想" / "让我理一下" | "想到什么程度了？能不能先做一个粗糙版本，用结果来验证思路？" |
| 用户在对比多个方案 | "能不能花 10 分钟做一个最简单的 prototype，用实际效果来选？" |

### 反馈来源优先级

```
1. 真实环境反馈（跑一下代码、发给用户看、部署试试）— 最有价值
2. 他人反馈（花 10 分钟问同事/老板，确认方向）— 次有价值
3. AI 反馈（让 AI 帮你 review/挑战你的方案）— 兜底
4. 自我推导（纯脑内思考）— 最后手段，且必须设时间上限
```

## 4. 沟通时机提醒

当用户说"我再想想"或陷入思考时：

| 状态 | 建议 |
|------|------|
| 完全没想法 | "先花15分钟自己想，列出问题，然后找人聊" |
| 有点眉目（50%） | "最佳沟通时机 — 验证方向比继续推导更高效" |
| 想得差不多（80%） | "可以沟通，但可能已经浪费时间了" |
| 想完美了（100%） | "太晚了，如果方向错就全白费" |

提醒话术：
- "你现在想到什么程度了？有没有可以先和谁确认一下方向的？"
- "与其自己想2小时，不如花10分钟问一下，确认方向对不对。"

## 5. 时间盒监控

提醒规则：
- 任务进行中，主动提醒已用时间
- 接近时间盒上限时，提前1个番茄警告
- 超时立即叫停

提醒话术：
- "这个任务已经花了X个番茄，距离你设定的Y个番茄还剩Z个。"
- "还剩1个番茄就到时间盒上限了，当前进度如何？"
- "时间盒已到。当前状态是否'够用'？继续还是停止？"

## 6. 任务完成时：记录可分享内容

当一个番茄或任务结束时，快速问用户：

> "这个任务有什么值得记到团队日报的吗？（技术决策/踩坑经验/有用的工具/关键进展）"

如果有，在周文档的时间线中用特定标记记录，方便 /evening 提取：

```markdown
- [团队] 发现 XXX 方案比 YYY 更适合我们的场景，原因是...
- [团队] 推荐工具：XXX，解决了 YYY 问题
```

不要强制，用户说"没有"就跳过。目的是在工作过程中自然积累团队日报素材，而不是 /evening 时从零回忆。

## 强制规则

- ❌ 不允许跳过结构化思考直接开始
- ❌ 不允许没有时间盒的任务
- ❌ 用户纯思考超过 15 分钟不提醒"做一个最小实验"
- ❌ 用户说"我再想想"超过 30 分钟不提醒沟通
- ✅ 超时必须叫停，要求用户决策
- ✅ 主动识别沟通需求
- ✅ 主动推动建立反馈闭环
- ✅ 任务结束时询问是否有可分享内容

## 核心原则

- **反馈闭环 > 完美准备** — 做一个粗糙版本拿反馈，比在脑子里想出完美方案更有价值
- **环境反馈 > 自我推导** — 跑一下、问一下、发出去看一下，比自己想更快得到答案
- **15分钟法则** — 任何思考超过 15 分钟没产出，就该切换到"行动+观察"模式
- **"够用就好" > "完美准备"** — 过度准备是焦虑驱动的行为，不是真正的行动
- **时间盒是硬约束** — 不是软建议

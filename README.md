# LessNote - AI 驱动的个人知识操作系统

> **Core Philosophy**: Force action, not optimize collection. 知行合一。

## 这是什么？

LessNote 是一个 AI 驱动的个人知识管理系统，核心理念是**用 AI 推动行动，而不是优化信息收集**。

它不是一个笔记工具，而是一个**认知操作系统**：
- AI 作为 Critical Thinking Partner，主动挑战你的认知偏见
- Skills 定义工作节奏（morning → focus → sync → evening）
- 目标对齐机制确保每日行动服务于长期目标
- 与团队日报仓库（iie-worklog）集成，个人记录自动提取为团队可见内容

## 使用这套体系，你能得到什么？

| 目标 | 怎么实现 |
|------|---------|
| **每天有清晰的 TOP 3** | `/morning` 自动汇总焦点、灰犀牛、超期任务，确认今日计划 |
| **减少无效思考，加速行动** | `/focus` 强制结构化 + 反馈闭环（ReAct 模式），15分钟无产出即提醒切换到行动 |
| **每周有可见的产出物** | 周文档强制要求"本周交付物"，`/evening` 每天检查焦点进度 |
| **AI 主动挑战你的拖延模式** | CLAUDE.md 中写入你的认知偏见，AI 识别到模式时 push back |
| **团队日报不再是负担** | 工作过程中自然积累素材，`/evening` 一键提取生成团队日报 |
| **长期目标不跑偏** | 年度目标 → 本周焦点 → 今日 TOP 3，每一层都有对齐检查 |

## 常见工作流

### 工作流 1：日常工作节奏

这是最核心的工作流，覆盖一天的完整周期。

```
09:00  /morning
       ├── AI 汇总：本周焦点、灰犀牛、超期任务、团队动态
       ├── 确认今日 TOP 3 + 时间盒
       └── 写入周文档 + 可选同步团队日报（今日计划）

09:30  /focus 写调研文档
       ├── 5 个结构化问题（问题、思路、沟通需求、完成标准、第一个里程碑）
       ├── 启动番茄钟
       ├── 执行中：每 15 分钟反馈闭环检查
       └── 完成时：有什么值得记到团队日报的？

12:00  午休

13:00  /sync
       ├── 评估上午进展 vs 今日 TOP 3
       ├── 调整下午计划
       └── 启动下一个番茄

13:30  /focus 跑通评测流程
       └── ...

18:00  /evening
       ├── 更新周文档时间线（完成/未完成）
       ├── 检查本周焦点进度
       ├── 简短回顾 + 明日预告
       ├── 一键生成团队日报 → git commit → 可选 push
       └── 作息提醒
```

### 工作流 2：个人记录 → 团队日报

解决"写日报是额外负担"的问题。核心思路：**在工作过程中自然积累素材，而不是下班后从零回忆。**

```
工作过程中（/focus）
  ├── 正常记录在周文档时间线中
  ├── 有值得分享的内容用 [团队] 标记
  │   例：- [团队] 发现 XXX 方案更适合，原因是...
  │   例：- [团队] 推荐工具：YYY，解决了 ZZZ 问题
  └── AI 在每个番茄结束时轻提醒

下班时（/evening）
  ├── AI 从时间线中提取 [团队] 标记 + 工作相关内容
  ├── 自动过滤个人事务（健康/理财/生活等）
  ├── 生成标准格式团队日报
  ├── 写入 iie-worklog 仓库
  └── git commit + 可选 push
```

**团队日报格式**：
```markdown
# YYYY-MM-DD 工作记录

## 今日工作
- [已完成] XXX
- [进行中] YYY

## 关键决策/发现
- 发现 AAA 方案比 BBB 更适合...

## 遇到的问题
- CCC 问题，需要 DDD 协助

## 推荐
- 工具/文章/想法

## 明日计划
- ...
```

### 工作流 3：周度复盘

每周一 /morning 自动触发周度检查：

```
周一 /morning
  ├── 创建新的周文档
  ├── 回顾上周焦点完成情况
  ├── 扫描项目全景：有无需归档/新增的项目
  ├── 检查想法收集箱：超过4周未决策的想法 → 处理或放弃
  └── 设定本周焦点（每件事必须有具体交付物）
```

### 工作流 4：深度思考

当面对需要深度分析的问题时，使用 `ai-prompts/` 中的提示词：

```
技术方案选型 → 圆桌讨论（多角色辩论）
概念理解     → 撕考者（掰开揉碎找骨架）
社会现象     → 沉思者（批判性深度思考）
决策分析     → principles/结构性思考框架.md（DECIDE 框架）
行为问题     → principles/福格行为模型.md（B = MAP 诊断）
```

## 快速开始

### 1. Clone 这个仓库

```bash
git clone <this-repo> ~/lessnote
cd ~/lessnote
```

### 2. 自定义核心文件

**必须自定义的文件：**

| 文件 | 做什么 |
|------|--------|
| `CLAUDE.md` | 写入你的工作上下文、团队信息、认知偏见挑战列表 |
| `goals/目标与原则参考.md` | 填写你的核心身份、价值观、职业方向 |
| `goals/年度目标-示例.md` | 替换为你的年度 OKR |
| `backlog/灰犀牛事项.md` | 替换为你的待办事项 |

**可以直接使用的文件：**
- `goals/认知误区与陷阱.md` — 通用认知偏见清单（选 3-5 条写入 CLAUDE.md）
- `.claude/skills/` — 工作节奏 Skills（可微调）
- `principles/` — 结构性思考框架 + 福格行为模型
- `ai-prompts/` — 圆桌讨论、撕考者、沉思者等提示词
- `backlog/想法收集模板.md` — 想法评估模板

### 3. 配置团队日报（可选）

如果你的团队有共享日报仓库（如 iie-worklog）：

1. 在 CLAUDE.md 中配置仓库路径和分支策略
2. morning skill 会自动读取团队成员日报
3. evening skill 会提供一键生成团队日报的功能

### 4. 配合 Claude Code 使用

```bash
cd ~/lessnote
claude

# 每天的四个节点
> /morning              # 启动：汇总 + 确认 TOP 3
> /focus 写调研文档      # 聚焦：结构化 + 时间盒 + 反馈闭环
> /sync                 # 同步：评估 + 调整
> /evening              # 总结：更新时间线 + 团队日报
```

### 5.（可选）配合任务管理 MCP 使用

LessNote 设计为与任务管理工具（如 Vibeflow MCP）配合使用：
- **任务管理 MCP**: 任务执行层（今日任务、番茄钟、状态追踪）
- **LessNote**: 认知反思层（目标对齐、行为模式、洞察记录）

没有任务管理 MCP 也能独立使用，Skills 中的 MCP 调用会自动跳过。

## 核心设计理念

### CLAUDE.md = AI 的长期记忆

`CLAUDE.md` 是整个系统的核心。它让 AI 成为 Critical Thinking Partner：
- **认知偏见挑战**：AI 识别到行为模式时，主动 push back
- **工作哲学注入**："行动产生信息"、"反馈闭环优先"
- **目标对齐**：每次对话都锚定长期目标

### Skills = 工作节奏固化

Skills 把最佳工作实践变成可重复执行的流程：
- `/morning`：汇总全局 + 团队动态 + 确认 TOP 3
- `/focus`：结构化思考 + 反馈闭环 + 时间盒 + 积累日报素材
- `/sync`：中间同步 + 评估调整
- `/evening`：更新时间线 + 焦点检查 + 一键团队日报

### 行动 > 收集

这个系统的每个设计决策都服务于一个目标：**推动你去行动**。
- 想法收集箱的目标是"评估和放弃"，不是"存储"
- 每日自查只有4个问题
- 认知偏见清单不是用来"理解自己"，是用来"改变行为"
- 反馈闭环（ReAct 原则）：15分钟无产出就该切换到行动模式

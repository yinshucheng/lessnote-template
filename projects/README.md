## 项目管理方法论

### 项目文档标准结构

每个项目目录下应包含 README.md，格式见：

projects/示例项目/README.md

### 停滞规则

- 项目超过 4 周无进展 → 归档到 `已完成/` 或标注"暂停"
- 每周一 `/morning` 检查活跃项目状态

### 与 AI 协作

当需要在某个项目的代码仓库中工作时，可以说：

```
> 帮我到 [项目名] 看一下最近的改动
> 切到 [仓库名] 目录，帮我跑一下测试
> 从 [项目名] 的代码里提取今天的工作进展，写到周文档
```

AI 会根据仓库速查表找到本地路径，直接在对应目录工作。

## 学习建议

1. **先用起来**：直接把感兴趣的 prompt 贴给 Claude 试，体验比阅读更有效
2. **学写法不学内容**：李继刚的 Lisp 风格展示了"用代码结构约束 LLM"的思路，这个思路可以用在 Skills 和 CLAUDE.md 设计中
3. **商用 system prompt 是最佳教材**：看 awesome-system-prompts 里泄露的真实 prompt，学习产品级 prompt 的结构和技巧
